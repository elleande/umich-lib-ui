<Text lede>The API documentation of the Alert React component.</Text>

Screenreader friendly alert messages. In many apps developers add "alert" messages when network events or other things happen. Users with assistive technologies may not know about the message unless you develop for it.

The Alert component will announce to assistive technologies whatever you render to the screen. If you don't have a screenreader on you won't notice a difference between rendering `<Alert>` vs. a `<div>`.

- Source: https://github.com/mlibrary/umich-lib-ui/blob/master/packages/alert/src/index.js

```js
import { Alert } from '@umich-lib/core'
```

```jsx live=true
<Alert>Hello there.</Alert>
```

## Alert props

|Prop|Type|Required|Default|
|----|----|--------|-------|
|[intent](#intent)|enum "informational", "warning", "error", or "success"|false|"informational"|
|[children](#children)|node|true|n/a|

### intent

*Type*: enum "informational", "warning", "error", or "success" default: "informational"

Controls how the `Button` is styled visually.

```jsx live=true
<Alert intent="error">It's too late.</Alert>
```
```jsx live=true
<Alert intent="success">You did it!</Alert>
```
```jsx live=true
<Alert intent="warning">Don't do it!</Alert>
```

### children

*Type*: `node`

Regular React children.

```jsx live=true
<Alert>
  <div 
    style={{
      display: 'flex',
      justifyContent: 'space-between',
      alignItems: 'baseline'
    }}
  >
    <p>Did you know?</p>
    <Button small onClick={
      () => alert('Clicked!')
    }>Yeah</Button>
  </div>
</Alert>
```
