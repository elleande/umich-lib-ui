## Link props

|Prop|Type|Required|Default|
|----|----|--------|-------|
|[kind](#kind)|enum "default", "subtle", "list", "list-strong", "special", "special"|false|"default"|
|[children](#children)|string|true|n/a|
|[element props](#element-props)|spread|false|n/a|

### kind

*Type*: enum "default", "subtle", "light", "special", "list", "list-medium", "list-strong", "description"

Controls how the `Link` is styled.

```jsx live
<Link>Example link</Link>
```

```jsx live
<Link kind="subtle">Example link</Link>
```

```jsx live
<div style={{
  background: COLORS.teal['400'],
  padding: SPACING['M']
}}>
  <Link kind="light">Example link</Link>
</div>
```

```jsx live
<Link kind="list">Example link</Link>
```

```jsx live
<Link kind="list-medium">Example link</Link>
```

```jsx live
<Link kind="list-strong">Example link</Link>
```

```jsx live
<Link kind="special">Example link</Link>
```

### children

*Type*: `node`

Regular React children.

```jsx live=true
<Link href="#"><strong style={{ fontWeight: '700' }}>All events</strong></Link>
```

### element props

Any props not listed above will be spread onto the underlying element.

```jsx live=true
<Link
  style={{ fontWeight: '700' }}
  href="#"
>
  Default link with other props
</Link>
```

## Link styles

If you need to use a component other than the anchor element, such as [Gatsby's Link](https://www.gatsbyjs.org/docs/gatsby-link/) or a routing library link, you can use `LINK_STYLES` with [Emotion](https://emotion.sh/docs/) and apply them.

```js
import { LINK_STYLES } from '@umich-lib/core'
```

```jsx
<a href="#" css={{ ...LINK_STYLES['special'] }}>Special link</a>
```
